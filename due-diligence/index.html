01

Executive summary

Three things an investor needs to believe: it works, it's cheap, and the output is safe to put in front of a paying client's audience. Here is the measured position on each.

It works — and it now targets multiple ICPs from one brief. All ten content tools were fired against the live production backend and every one returned real, on-brief marketing content (HTTP 200). A full Content Pipeline launch — deep ICP research, then a full asset set generated in parallel where every asset is derived from that ICP's strongest ranked fear/desire — completes end-to-end, unattended, in about five minutes. Six consecutive launches (3 single-ICP, 3 multi-ICP) all completed; four one-shot and two via the auto-resume path — the reliability defect that required the resume is diagnosed and fixed (§Reliability).

It's cheap — and the number is recorded, not estimated. Every generation writes its true dollar cost to a usage_ledger row. A single-ICP launch (9 assets, incl. VSL + AI image) measures $1.02 (range $0.98–1.07). A multi-ICP launch targeting 3 distinct ICPs (19 assets) measures $1.93 — i.e. tripling the audience coverage costs only ~1.9×, not 3×, because each extra ICP adds five fast text assets, not a whole launch. The investor's working estimate of $15–50 per launch is still 15–48× too high. The only genuinely premium cost in the stack is the VSL (long-form Opus, ~$0.18/run), correctly gated to the top tier and now token-bounded so a launch always completes in one pass.

The output is governed, not hoped-for. Every tool's output runs through an adversarial three-judge certification (copy quality, factual honesty, formatting) and no asset reaches a client without passing a human QC gate. That certification did its job: it caught a real fabrication risk in seven tools, and a hard anti-fabrication guardrail was written and deployed to production the same day. Re-certification of the fix is the one remaining open item (§09).

The honest headline. This is not a "trust us, it's great" deck. Every number below is tagged with how it's known and carries the exact query to reproduce it. The one place we found a problem, we say so — and show the fix.

05

Quality methodology & the fabrication finding

"World-class output" is a claim that has to survive an adversary. So every output is graded by three independent judges, each trying to fail it on a different axis — and the pipeline is proven on all three surfaces a client can use.

Three surfaces, three judges

Surface coverage

The same generators power all three ways a client is served — standalone tools, the Content Pipeline, and the AI Chat agent — so a fix in one is a fix in all three.

Judge A — Copy

Persuasion, specificity, brand voice. Verdict: all tools ≥ 8/10. World-class.

Judge C — Format

Structure, clean rendering, no stray tokens or artifacts. Verdict: all tools ≥ 8/10. Clean.

Judge B — Factual honesty — caught a real problem. Seven of ten tools were inventing proof that does not exist: fabricated testimonials ("— Dave, DH Heating, Essex"), made-up case-study numbers ("47 leads in 28 days"), invented aggregate proof ("200+ firms", "4.9★"), and unbackable guarantees. For a real client-facing business this is not a style nit — fabricated guarantees and testimonials create direct client legal exposure. Only ICP Research and Disclaimers were clean.
Fixed the same day, at the root. A hard NO-FABRICATION guardrail was written into the shared formatting rules injected by nine generators — and directly into the landing-page prompt, which was the worst offender. It forbids invented testimonials, statistics, client counts, ratings and guarantees, and forces any illustrative example to be framed unambiguously as hypothetical. A British-English + current-year locale rule shipped with it. All affected generators were redeployed to production. The finding, the fix, and the deploy are documented; re-certification to prove zero fabrication is the final open gate (§09).

The gate that makes it safe regardless

Even before that fix, no unreviewed asset can reach a client. Every launch lands qc_status = 'in_review'; the client share link shows a "being finalised" card until a human approves; and a client-segmented Review Dashboard (Kanban: In review → Rerunning → Approved) lets the operator approve good assets and re-run only the ones that need work — the approved ones are never touched. Output is also auto-scanned by validateOutput(), which holds anything dirty.

09b

Known limitations & where the evidence is thin

Stated plainly, so diligence finds nothing we haven't already named.

Evidence breadth

  • Sample = 6 launches, one independent judge, asset openings. Quality is graded from strong openings, not full-body reads across a large denominator. Roadmap: 20+ full launches, a multi-judge panel, full-asset grading.
  • Reliability = 4/6 one-shot (2 completed via auto-resume). Root cause found and fixed (VSL token cap); a broader concurrent load test is to be re-run.
  • Fabrication re-cert pending (§09) — the guardrail is shipped, but the exhaustive all-tools re-test has not yet been run.

Defensibility & blended economics — the honest answer

  • No data moat. The stack uses off-the-shelf frontier models (gpt-5.5 / Opus); there is no proprietary training or network effect. The defensible edge is execution — the ICP-derivation workflow, operator/QC tooling, integration into a done-for-you service, and speed to a full multi-ICP campaign — not the per-prompt token curve.
  • $1.02 is the API cost, not the fully-loaded cost. It excludes rendered VSL video (a premium-tier item, ~$10–40/render) and human QC labour (~5–10 min/launch). Blended per-launch economics at scale are dominated by those two — both isolated to the premium tier by design.
