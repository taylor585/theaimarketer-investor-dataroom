02

How the audit run works

A self-running generate → capture → grade → fix loop. Every tool is exercised fresh on all three surfaces a client can be served, graded on seven axes, and — where it misses — the generator is fixed and re-run until it passes.

3 surfaces

Every tool is fired standalone, inside a full Launchpad run, and via AI Chat — proving quality holds however a client is served.

Fresh, post-fix

Claude drives the live app and clicks Generate / Launch itself; nothing is graded on stale pre-fix assets.

Test & fix

Below bar → wire the fix into the generator, redeploy (with your approval), re-fire, re-grade. The loop improves outputs, not just measures them.

The seven grading axes (0–10, world-class = ≥8)

A
ICP-derivation — traces to the ICP's strongest fears/desires; smart selection & distribution. primary
B
Prompt-fidelity — delivered what its own generation prompt demanded.
C
Copy craft — hooks scroll-stopping; hard-hitting; storyline/flow constructed perfectly (hook→problem→mechanism→proof→CTA).
D
Honesty — no invented testimonials, stats, or guarantees.
E
Formatting & brand — clean structure, on-brand, no artifacts.
F
Visual quality — images flaw-free (hands/faces/text/warping) and premium; landing renders live.
G
Access-ease — clicks-to-reach; Copy / Download / Share actually work.

Capture matrix & stop rule

10 tools × 3 surfaces = 30 fresh captures + the full-launch gallery + the QC board. The loop stops only when every cell is captured, graded, screenshotted, the Derivation Matrix is complete, and an adversarial completeness-critic finds no gap — or a cell errors twice (→ logged as a real bug).

How it works now (from the code): ICP research emits a ranked "Top Angles" list (fear/desire, strongest first). _shared/icpSelection.ts selects the strongest N per ICP; each generator injects the Master ICP-Derivation Instruction and builds one asset per distinct angle. The former failure mode — generators getting a raw ICP dump with no ranking, and generate-ai-images getting no ICP at all — is fixed and deployed across all generators.

The mechanism — the Master ICP-Derivation Instruction (shipped in every generator)

# prepended to each craft generator's prompt — DEPLOYED to all generators on mebct
Before writing, read the ICP and extract FEARS, DESIRES, OBJECTIONS.
Rank each by intensity + commercial pull. Then SELECT the strongest
angles for THIS asset and DISTRIBUTE them — never reuse one angle or
default to generic pain:
  • N images / N ad variations → the 2 strongest fears + 3 strongest
    desires, one dominant angle per asset
  • Hooks → spread across the top angles; no two on the same one
  • Email / VSL / video → open on the single strongest fear, escalate
    through desire, resolve on the offer
Write in the audience's own words. Apply world-class direct-response
craft: pattern-interrupt hook → problem agitation → mechanism →
honest proof → one crisp CTA. Never invent testimonials/stats/guarantees.
Tag the ICP angle each asset targets (internal) so selection is auditable.

Wire into

generate-ad-copy, -hooks, -static-images, -video-scripts, -vsl-script, -email-sequence, -landing-pages.

Plus images

Pass the selected angle + a quality/negative prompt into generate-ai-images so the picture matches the chosen fear/desire.
